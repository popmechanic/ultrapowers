## Judgments (keys joined after judging)

Blinding: all three prompts assigned X=B, Y=A (independent per-process seeds;
1-in-8 coincidence). Verdict files: `judgment-{wide,chained,webapp}.json`.

- **wide → A.** Implementations functionally identical; A's tests a strict
  superset (ngrams `n==len`/unigram/empty-list, reverse_words single-word +
  tab/newline, word_count case-folding). B: 2 missing-edge-test defects, A: 0.
- **chained → A.** B's `format_cents` does money math in binary floating point
  (the one latent correctness defect found in any diff), plus a 200k-iteration
  scope-creep stress test, CRLF fragility in the CLI, and comment noise (5
  defects). A: 3 minor coverage/leniency nits.
- **webapp → B.** A rewrote the npm test script to `node --test "tests/**/*.test.js"`
  — the quoted glob is not expanded and Node 20 (the plan's declared minimum) does
  not glob `--test` args, breaking the whole acceptance invocation; A also parses
  the body before routing (400 where the spec requires 404) and can double-respond
  on stream errors (4 defects, two substantive). B: 4 minor defects (store.list()
  returns internal array by reference, unauthorized-but-equivalent test-script
  edit, one missing case, one near-vacuous test).

## Diagnosis notes (for the follow-up, not a rationalization of the miss)

- **No causal path from the subtraction to the misses was found.** The deleted
  machinery (standalone prose-pin tests, complexity ratchet, compiler near-miss
  diagnostics, `waveLabels`, sealed-by-default routing) does not touch implementer
  prompts, task bodies, tiering, review depth, or the merge/gate path. The judged
  defects (missing edge tests, float math, a bad glob) are implementer-output
  variance of a kind seen in both engines' diffs.
- B's chained overrun (+258s, +17k tokens) is largely a review-caught fix round —
  the machinery working, but paid for in the same run; A's chained run happened to
  need none. B's webapp overrun (+33k tokens) bought a cleaner outcome (PASS, 0
  acks, minor-only defects) where A shipped a suite-breaking script edit.
- **Recommended next step:** rerun the two out-of-band cells (chained, webapp)
  once per engine (4 cells, ~1 hour) to get n=2 on the divergent fixtures before
  concluding an engine-caused regression. If B's token overrun and the quality
  split reproduce, diagnose for real; if they wash out, re-run the tag-rule check
  on the pooled rows.
